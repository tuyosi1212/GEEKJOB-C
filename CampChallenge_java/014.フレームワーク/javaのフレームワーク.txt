Spring Framework
特徴
・できるだけプログラムを簡潔に書くことを目指している。
・シンプルな構成、軽量のマイクロフレームワーク
・モジュール次第で幅広いジャンルのアプリやソフトの開発に利用できる。
　（I/Oプロジェクト、アプリケーション開発、webアプリ開発、クラウド上のソフト開発、データベース管理など）
環境構築
１、Spring Frameworkインストール
２、STS(Spring Tool Suite)インストール
３、プロジェクト作成、ビルド（Maven)
４、サーバ（STS)にプロジェクト追加
５、サーバ起動
６、実行結果を確認


Play Flamework
特徴
・ScalaというJavaと親和性の高い言語にも使用可能
・軽量で高い生産性
・javaのweb使用アプリケーション標準仕様に準拠しておらず、ServletやJSPを使わず新しいアプローチでのアプリケーション構築を行える。
環境開発
１、Play Flamework インストール
２、JDK,sbtの環境構築（sbtはPlay Flameworkの環境構築に必要）
３、Eclipseをインストール・環境設定
４、環境変数PATHを設定
５、コマンドプロントにて実行しPlayがインストールされることを確認
６、アプリを起動し起動できるか確認（質問などがある。Enterなど押す）
７、Eclipseにインポート


Java EE　（2月にJakarta EEに名称変更）
特徴
・大規模システム向け、企業向け
・Java標準仕様（Servlet、JSPなど）
・近年使いやすさや生産性が向上　
環境構築
１、Java EE７をインストール
２、Eclipseをインストール・環境設定
３、GlassFIsh serverの追加
４、プロジェクト作成（Java EEの）
５、Eclipseに取り込む
６、実行する
